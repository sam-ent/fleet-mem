# Support

fleet-mem is maintained by a single developer in spare time. I appreciate your patience.

## Getting help

| Channel | Use for |
|---|---|
| [GitHub Discussions](https://github.com/sam-ent/fleet-mem/discussions) | Questions, how-to, ideas |
| [Bug reports](https://github.com/sam-ent/fleet-mem/issues/new?template=bug_report.yml) | Confirmed bugs with reproduction steps |
| [Feature requests](https://github.com/sam-ent/fleet-mem/issues/new?template=feature_request.yml) | Well-defined enhancement proposals |

## Response times

This is a best-effort, volunteer project. There is no SLA.

- **Bugs with reproduction steps**: prioritized
- **Feature requests**: reviewed when capacity allows — accepted features get the `accepted` label
- **Questions**: please use Discussions; issues opened as questions will be converted or closed

## What won't get a response

- Issues that don't follow the template
- "Me too" comments without new information
- Requests for general Python/AI/LLM help unrelated to fleet-mem
